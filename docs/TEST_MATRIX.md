# ApexPlayer 轻量验收矩阵（D2）

## 1. 验收环境
- 平台：macOS 13+
- 构建：`swift build` / `swift test`
- 音乐样本：>= 200 首（混合 mp3/m4a/flac/wav/aiff）

## 2. 功能验收用例
1. 目录导入
- 步骤：添加包含 200 首文件的目录。
- 期望：曲库可见，扫描状态显示进度并最终完成。

2. 手动文件导入
- 步骤：手动选择多个文件导入。
- 期望：新增曲目进入列表，元数据正常显示。

3. 基础播放控制
- 步骤：播放/暂停/上一首/下一首/拖动进度。
- 期望：状态一致，进度更新准确。

4. 播放模式
- 步骤：分别测试顺序/随机/单曲循环。
- 期望：曲目自然结束后按模式续播。

5. 播放列表
- 步骤：创建、重命名、删除、添加曲目、拖拽排序、多选删除。
- 期望：所有操作立即生效，重启后保持。

6. 收藏与最近播放
- 步骤：收藏/取消收藏，播放后查看最近播放页面，按天分组显示。
- 期望：展示正确；清空历史后列表为空。

7. ReplayGain 与淡入淡出
- 步骤：切换 ReplayGain，调整淡入淡出时长。
- 期望：播放音量和过渡行为符合设置。

8. 系统媒体键
- 步骤：使用键盘媒体键控制。
- 期望：播放器能响应，Now Playing 信息同步。

9. 连续播放稳定性
- 步骤：连续播放 30 分钟。
- 期望：无崩溃，无明显内存暴涨。

10. 恢复上次播放
- 步骤：播放后退出重启。
- 期望：恢复到上次曲目与进度（暂停态）。

## 3. 通过标准
- 上述 10 项全部通过，或仅允许 1 项低风险 UI 级问题（不影响播放与数据持久化）。
