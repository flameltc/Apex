# ApexPlayer 性能基线记录

## 1. 目的
- 在每次发布前记录导入性能，快速识别回归。

## 2. 测试环境
- macOS:
- 机器型号:
- CPU/RAM:
- 构建模式: release

## 3. 数据集要求
- 数据集 A: 200 首混合格式（mp3/m4a/flac/wav/aiff）
- 数据集 B: 1000 首混合格式
- 数据集 C: 2000 首（可选）

## 4. 执行方法
1. 启动应用并清空旧库（可选）
2. 导入对应目录
3. 记录从“开始导入”到“导入完成”的耗时
4. 重扫一次并记录耗时

## 5. 记录模板
| 日期 | 数据集 | 导入耗时 | 重扫耗时 | 备注 |
|---|---|---:|---:|---|
| YYYY-MM-DD | A(200) |  |  |  |
| YYYY-MM-DD | B(1000) |  |  |  |
| YYYY-MM-DD | C(2000) |  |  |  |

## 6. 回归判定建议
- 相比上次基线，任一数据集导入或重扫耗时增加 > 25%，视为性能回归，需要排查。
